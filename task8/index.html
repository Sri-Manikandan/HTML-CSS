Frequently Asked Questions

CSS Grid is a two-dimensional layout system that lets you arrange elements in rows and columns simultaneously. Use it when you need precise control over both axes — for example, full-page layouts, card galleries, or any design where items need to align across rows and columns. Grid shines when your layout is defined by the container rather than the content.
Flexbox is a one-dimensional layout model — it handles either a row or a column at a time. It is ideal for distributing space among items in a single axis, such as navigation bars, button groups, or centering content. Grid handles two dimensions simultaneously and is better suited for complex page-level structures. The two are complementary and often used together.
CSS custom properties, also called CSS variables, let you store values under a name prefixed with two dashes, such as --primary-color. You declare them on an element (typically :root for global scope) and reference them with var(--primary-color). Unlike preprocessor variables, they are live in the browser, can be overridden in any scope, and are accessible from JavaScript, making them powerful for theming and dynamic styling.
Every element in CSS is represented as a rectangular box consisting of four layers: content, padding, border, and margin. By default, width and height apply only to the content area, so padding and border add to the total size. Setting box-sizing: border-box makes width and height include padding and border, which is far more predictable and is why it is included in most CSS resets today.
Transitions respond to state changes — they interpolate a property from its current value to a new value when triggered by an event like hover or focus. You define the property, duration, easing, and delay. Animations, defined with @keyframes, run independently of state changes and can loop, alternate, and contain multiple steps. Use transitions for simple interactive feedback and animations for choreographed sequences that run on their own.
Specificity determines which CSS rule applies when multiple rules target the same element. It is calculated as a three-part score: ID selectors contribute 1-0-0, class, attribute, and pseudo-class selectors contribute 0-1-0, and type selectors and pseudo-elements contribute 0-0-1. The selector with the highest score wins. Inline styles outrank all selectors, and !important overrides everything. Keeping specificity low and consistent — typically by relying on classes — makes stylesheets easier to maintain.
